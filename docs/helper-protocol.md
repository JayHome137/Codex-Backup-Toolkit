# Local Helper Protocol

This document describes the draft protocol between the Web GUI and a future native/local helper.

The current Web GUI does **not** execute shell commands. `Local Bridge` mode uses this protocol shape with a mock helper transport so the safety model can be tested before a native helper exists.

## Safety Boundary

The helper protocol is allowlist-first. A request can only be created for:

- `doctor`: `./scripts/codexbackup.sh --doctor --target <target>`
- `validate`: `./scripts/codexinstallautomation.sh validate` with `CODEX_BACKUP_LAUNCHD_LABEL=dev.codexbackup.toolkit.test.<target>`

The GUI blocks backup, restore, install, uninstall, status, and any command outside the allowlist. The isolated validate label is intentional: it prevents the GUI from touching a user-installed backup job.

## Request Schema

```json
{
  "schema": "codex-backup-helper.v1",
  "version": 1,
  "requestId": "cbt_<generated>",
  "createdAt": "2026-06-04T00:00:00.000Z",
  "kind": "doctor",
  "command": "./scripts/codexbackup.sh --doctor --target local"
}
```

Fields:

- `schema`: fixed protocol name, currently `codex-backup-helper.v1`.
- `version`: numeric protocol version, currently `1`.
- `requestId`: generated client-side id for correlation and audit logs.
- `createdAt`: ISO timestamp generated by the GUI.
- `kind`: `doctor` or `validate`.
- `command`: the exact command preview that passed the allowlist.

## Response Schema

```json
{
  "schema": "codex-backup-helper.v1",
  "version": 1,
  "requestId": "cbt_<generated>",
  "status": "ok",
  "exitCode": 0,
  "stdout": "Doctor passed.",
  "stderr": "",
  "audit": {
    "commandKind": "doctor",
    "decision": "allowed",
    "helper": "native-helper",
    "startedAt": "2026-06-04T00:00:01.000Z",
    "finishedAt": "2026-06-04T00:00:01.500Z"
  }
}
```

Fields:

- `status`: `ok` or `error`.
- `exitCode`: process-style exit code. For mock helper success this is `0`.
- `stdout`: command output or helper message.
- `stderr`: command error output when available.
- `errorCode`: optional machine-readable error code.
- `audit`: required audit summary for every helper response.

## Error Codes

- `ERR_COMMAND_NOT_ALLOWED`: command failed the GUI allowlist before reaching the helper.
- `ERR_HELPER_UNAVAILABLE`: GUI could not reach or use the helper transport.
- `ERR_HELPER_FAILED`: helper responded but the command failed.

## Current Implementation

The current implementation lives in the Web GUI code:

- `gui/src/lib/localBridge.ts`: allowlist classification and Local Bridge runner.
- `gui/src/lib/helperProtocol.ts`: request/response types, request builder, mock transport.

No native helper server is started by this repository yet. The mock transport exists so the UI can show request ids, schema names, command kinds, decisions, helper identity, and exit codes without running shell commands.
